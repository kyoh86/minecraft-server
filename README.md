# minecraft-server

このリポジトリは、Docker で Minecraft Java サーバー（Paper 1.21.11）を
運用するための構成を管理する。

サーバーに参加したい人は kyoh86 までご連絡を。
専用Discordサーバーに招待します。

## このサーバーの魅力

このサーバーは、ただ「マイクラを建てる場所」ではありません。
初参加のしやすさと長期運用の遊びやすさを両立するために、
参加者体験を中心に設計しています。

### 初参加が迷わない

- 未認証プレイヤーは `limbo` に誘導され、ワンタイムコードで連携できる
- 画面内でコードをコピーし、Discord の `/mc link` を実行するだけ
- 手順が短く、初心者でも参加しやすい

### ワールド分離で遊び方がぶつからない

- `mainhall`: 移動ハブ
- `residence`: 生活・建築
- `resource`: 資源採集
- `factory`: 自動化・装置運用

建築派、探索派、技術派が同時に遊んでも、目的の違いで衝突しにくい構成です。

### 資源枯渇に強く、アップデートにも強い

- `resource` は再生成前提で運用する
- 資源が減っても循環できる
- バージョンアップ後の新バイオーム・新地形にも早く出会える
- 生活拠点のあるワールドは維持されるため、資産を守りやすい

「資源はリフレッシュ、拠点は保全」を両立できます。

### 移動導線がわかりやすい

- すべてのワールド移動を `mainhall` に集約
- 往復ポータルを整備し、戻り先に迷いにくい
- 安全テレポート設定で移動事故を抑制

### 事故りにくい安心設計

- ゲート周辺は保護リージョンで建築破壊や爆発を抑止
- ログイン直後は安全範囲へ補正し、危険位置スポーンを防止
- 利用可否エリアをメッセージ表示し、ルールを体感で把握しやすい

### 運用の安定が参加者体験につながる

- 稼働監視と通知で異常検知を早める
- 接続先変更や参加/退出の通知導線を整備
- `mc-ctl` による手順化で運用ミスを減らす

自由に遊べるのに、荒れにくく、迷いにくく、長く続けやすい。
このサーバーは、参加者が快適に遊び続けるために設計された Minecraft サーバーです。
